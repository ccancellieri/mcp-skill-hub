Sandbox vs. real Chrome session

By default, chrome-devtools-mcp launches an isolated browser instance — no access to your personal tabs, history, or saved passwords.

To connect to a running Chrome window (with remote-debugging enabled), start Chrome with the flag below and configure the plugin accordingly: 

--remote-debugging-port=9222

Then pass --browserUrl=http://127.0.0.1:9222 to the MCP server. Real-profile launcher support is a planned future feature.
